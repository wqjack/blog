# 系统设计题库（前端角度：题 + 解法要点 + 加分点）

## 1 文件上传系统设计
**题目**：设计一个支持大文件断点续传的文件上传系统（前端角度）。  
**要点**：
- 分片上传（chunk）并支持并发上传  
- MD5 或 SHA256 去重（可选）  
- 后端合并并校验  
- 进度显示与失败重试策略  
**前端实现要点**：文件分片、并发控制、断点记录（localStorage/IndexedDB）、分块重试、hash 计算在 worker 中执行。  
**加分点**：秒传、断点续传 UX、并发优化、节省内存。

## 2 日志采集系统（前端埋点）
**题目**：如何设计高吞吐的前端埋点上报系统？  
**要点**：批量上报、压缩、降采样、使用 Beacon API / navigator.sendBeacon、链路追踪 traceId、异常上传策略。  
**加分点**：埋点 SDK 的可配置性、离线缓存、数据一致性处理。

## 3 权限系统（前端实现）
**题目**：前端如何实现基于角色的权限控制？  
**要点**：路由级别权限、按钮级权限、后端下发权限策略、前端缓存合理时效、动态路由加载。  
**加分点**：ABAC、策略中心、细粒度权限与缓存失效策略。

## 4 配置中心（前端灰度）
**题目**：如何通过配置中心做前端功能灰度？  
**要点**：远程配置拉取、开关中心、动态 feature flag、客户端缓存与刷新策略、回滚机制。  
**加分点**：实验平台（AB test）、可视化灰度面板、策略基于用户属性。

## 5 大型前端架构设计
**题目**：如何设计一个企业级前端架构（多团队协作）？  
**要点**：Monorepo + package 管理、组件库与设计系统、CI/CD、发布策略、代码分包、共享依赖治理。  
**加分点**：Module Federation 微前端策略、统一监控与异常收集、按团队拆分权限与模块。

## 6 实时协作（如在线编辑）
**题目**：如何实现多人实时协作的前端（冲突解决、同步）？  
**要点**：OT 或 CRDT 算法、操作合并、离线编辑策略、延迟补偿、权威合并策略。  
**加分点**：性能与带宽优化、光标显示、权限控制。

## 7 高并发页面渲染与性能
**题目**：如何保证一个列表页在数据量大并发访问时性能稳定？  
**要点**：虚拟列表（windowing）、分页/游标分页、后端聚合、CDN 缓存、前端缓存与去重。  
**加分点**：预取策略、增量更新、差分渲染实现细节。

## 8 搜索系统（前端接入）
**题目**：前端如何接入搜索系统并优化体验？  
**要点**：debounce + throttling、结果高亮、前端缓存、节流请求、分页与 infinite scroll。  
**加分点**：搜索建议（typeahead）、拼写纠正、并行多源检索聚合。

## 9 OTT 或视频平台（播放统计）
**题目**：设计一个视频播放统计系统，从前端视角讲解关键点。  
**要点**：心跳采集、播放进度上报、卡顿/缓冲上报、带宽估计、低成本批量上报。  
**加分点**：精确播放率计算、断点续播策略、观看质量分层。

## 10 面试常见系统设计题作答模版（结构化）
**模板**：场景 -> 目标（SLA）-> 约束 -> 高层设计 -> 关键细节（存储/缓存/一致性）-> 容错/扩展 -> 监控与指标 -> 安全与成本 -> 结论与权衡
